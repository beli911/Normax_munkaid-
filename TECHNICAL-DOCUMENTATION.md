## 🎯 Rendszer Áttekintés

### Fő Funkciók

- **Munkaidő Rögzítés**: Dolgozók rögzíthetik a napi munkaidejüket (dátum, kezdés, vége, megjegyzés). Éjszakás műszak (pl. 16:00–02:00) támogatott: „Másnap fejeződik be” checkbox, helyes óraszámítás és bérszámítás.
- **Admin Felület**: Adminok kezelhetik az alkalmazottakat és megtekinthetik az összes rögzítést
- **Export Funkciók**: CSV, Excel és JSON formátumban exportálható az adat
- **Biztonság**: Rögzítések nem szerkeszthetők és nem törölhetők (csak admin módosíthat)

### Technológiai Stack

- **Frontend**: Next.js 14 (React), Tailwind CSS
- **Backend**: Supabase (PostgreSQL + Auth + Real-time)
- **Deployment**: Vercel (ajánlott)

## 🗄️ Adatbázis Struktúra

### `profiles` Tábla

```sql
CREATE TABLE public.profiles (
  id uuid PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE,
  email text,
  full_name text,
  is_admin boolean DEFAULT false,
  hourly_rate numeric
);
```

**Mezők:**
- `id` - UUID, kapcsolódik az `auth.users` táblához
- `email` - Email cím (pl. `user@munkaido.local`)
- `full_name` - Teljes név (pl. "Kiss János")
- `is_admin` - Admin jogosultság (true/false)
- `hourly_rate` - Óradíj Ft/óra (pl. 1500)

**RLS Policy-k:**
- Dolgozók: csak a saját profiljukat látják
- Adminok: minden profilt látnak

### `timesheets` Tábla

```sql
CREATE TABLE public.timesheets (
  id bigint PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  user_id uuid REFERENCES public.profiles(id) NOT NULL,
  work_date date NOT NULL DEFAULT CURRENT_DATE,
  start_time time NOT NULL,
  end_time time NOT NULL,
  note text,
  created_at timestamp with time zone DEFAULT timezone('utc'::text, now()) NOT NULL
);
```

**Mezők:**
- `id` - Egyedi azonosító
- `user_id` - Kapcsolódik a `profiles.id`-hoz
- `work_date` - Munkanap (YYYY-MM-DD)
- `start_time` - Kezdési idő (HH:MM)
- `end_time` - Befejezési idő (HH:MM)
- `note` - Megjegyzés (opcionális)
- `created_at` - Létrehozás időpontja

**RLS Policy-k:**
- Dolgozók: csak a saját rögzítéseiket látják és hozhatják létre
- Adminok: minden rögzítést látnak
- **FONTOS**: Dolgozók NEM módosíthatják és NEM törölhetik a rögzítéseket

## 🔌 API Endpointok

### 1. Felhasználó Létrehozása

**Endpoint:** `POST /api/admin/create-user`

**Autentikáció:** Bearer token (admin jogosultság szükséges)

**Request Body:**
```json
{
  "username": "kovacs.janos",
  "password": "jelszo123",
  "fullName": "Kovács János"
}
```

**Response (201):**
```json
{
  "userId": "uuid",
  "email": "kovacs.janos@munkaido.local"
}
```

**Response (403):**
```json
{
  "error": "Nincs jogosultsag."
}
```

**Példa cURL:**
```bash
curl -X POST http://localhost:3000/api/admin/create-user \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -d '{
    "username": "kovacs.janos",
    "password": "jelszo123",
    "fullName": "Kovács János"
  }'
```

## 🔐 Autentikáció

### Bejelentkezés

A rendszer Supabase Auth-ot használ. A bejelentkezés email + jelszó alapú.

**Email formátum:**
- Ha a felhasználónév tartalmaz `@` karaktert → emailként használja
- Ha nem → hozzáadja a `@munkaido.local` végződést
- Példa: `kovacs.janos` → `kovacs.janos@munkaido.local`

**Ékezetes karakterek:**
- Automatikusan eltávolítódnak az email címből
- Példa: `Belián` → `belian@munkaido.local`

### Access Token

A bejelentkezés után az access token-t a Supabase session-ből kapod:

```javascript
const { data: { session } } = await supabase.auth.getSession()
const accessToken = session?.access_token
```

### Admin Jogosultság Ellenőrzése

Minden admin API endpoint ellenőrzi az admin jogosultságot:

```javascript
const { data: profile } = await supabase
  .from('profiles')
  .select('is_admin')
  .eq('id', user.id)
  .single()

if (!profile?.is_admin) {
  return NextResponse.json({ error: 'Nincs jogosultsag.' }, { status: 403 })
}
```

## 🔒 Biztonsági Beállítások

### Row Level Security (RLS)

Az adatbázis RLS policy-kkel védett:

**Profiles tábla:**
- Dolgozók: csak a saját profiljukat látják
- Adminok: minden profilt látnak

**Timesheets tábla:**
- Dolgozók: csak a saját rögzítéseiket látják és hozhatják létre
- Adminok: minden rögzítést látnak
- **Dolgozók NEM módosíthatják és NEM törölhetik a rögzítéseket**

### Public Signup

A public signup (nyilvános regisztráció) **KI VAN KAPCSOLVA**. Csak admin hozhat létre felhasználókat.

### Service Role Key

A felhasználó létrehozáshoz service role key szükséges (backend-only, soha ne használd frontend-en!).

## 💻 Példa Kódok

### JavaScript/TypeScript

```javascript
// 1. Supabase client inicializálása
import { createClient } from '@supabase/supabase-js'

const supabase = createClient(
  process.env.NEXT_PUBLIC_SUPABASE_URL,
  process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY
)

// 2. Bejelentkezés
async function login(username, password) {
  const normalized = username.toLowerCase().trim()
  const email = normalized.includes('@') 
    ? normalized 
    : `${normalized}@munkaido.local`
  
  const { data, error } = await supabase.auth.signInWithPassword({
    email,
    password
  })
  
  if (error) throw error
  return data
}

// 3. Munkaidő rögzítés létrehozása
async function createTimesheet(workDate, startTime, endTime, note = '') {
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) throw new Error('Nincs bejelentkezve')
  
  const { data, error } = await supabase
    .from('timesheets')
    .insert([{
      user_id: user.id,
      work_date: workDate,
      start_time: startTime,
      end_time: endTime,
      note
    }])
    .select()
    .single()
  
  if (error) throw error
  return data
}

// 4. Rögzítések lekérése (dolgozó)
async function getMyTimesheets() {
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) throw new Error('Nincs bejelentkezve')
  
  const { data, error } = await supabase
    .from('timesheets')
    .select('*')
    .eq('user_id', user.id)
    .order('work_date', { ascending: false })
  
  if (error) throw error
  return data
}

// 5. Összes rögzítés lekérése (admin)
async function getAllTimesheets() {
  const { data: { user } } = await supabase.auth.getUser()
  if (!user) throw new Error('Nincs bejelentkezve')
  
  // Ellenőrizzük az admin jogosultságot
  const { data: profile } = await supabase
    .from('profiles')
    .select('is_admin')
    .eq('id', user.id)
    .single()
  
  if (!profile?.is_admin) {
    throw new Error('Nincs admin jogosultság')
  }
  
  const { data, error } = await supabase
    .from('timesheets')
    .select('*, profiles(full_name, email, hourly_rate)')
    .order('work_date', { ascending: false })
  
  if (error) throw error
  return data
}
```

### Python

```python
from supabase import create_client, Client
from datetime import date, time

# Supabase client inicializálása
supabase: Client = create_client(
    os.environ.get("SUPABASE_URL"),
    os.environ.get("SUPABASE_ANON_KEY")
)

# Bejelentkezés
def login(username: str, password: str):
    normalized = username.lower().strip()
    email = normalized if '@' in normalized else f"{normalized}@munkaido.local"
    
    response = supabase.auth.sign_in_with_password({
        "email": email,
        "password": password
    })
    
    return response.session

# Munkaidő rögzítés létrehozása
def create_timesheet(work_date: str, start_time: str, end_time: str, note: str = ""):
    user = supabase.auth.get_user()
    if not user:
        raise Exception("Nincs bejelentkezve")
    
    response = supabase.table("timesheets").insert({
        "user_id": user.user.id,
        "work_date": work_date,
        "start_time": start_time,
        "end_time": end_time,
        "note": note
    }).execute()
    
    return response.data

# Rögzítések lekérése
def get_my_timesheets():
    user = supabase.auth.get_user()
    if not user:
        raise Exception("Nincs bejelentkezve")
    
    response = supabase.table("timesheets")\
        .select("*")\
        .eq("user_id", user.user.id)\
        .order("work_date", desc=True)\
        .execute()
    
    return response.data
```

## 📝 API Response Kódok

- **200 OK** - Sikeres kérés
- **201 Created** - Sikeres létrehozás
- **400 Bad Request** - Hibás kérés (pl. hiányzó mezők)
- **401 Unauthorized** - Nincs autentikáció
- **403 Forbidden** - Nincs jogosultság (pl. nem admin)
- **500 Internal Server Error** - Szerver hiba

## 🔍 Hibakeresés

### Gyakori Hibák

1. **"column profiles.hourly_rate does not exist"**
   - **Megoldás:** Futtasd le a `database/add-hourly-rate.sql` fájlt

2. **"Nincs jogosultságod"**
   - **Megoldás:** Ellenőrizd, hogy az `is_admin` mező `true`-e a `profiles` táblában

3. **"Unable to validate email address"**
   - **Megoldás:** A felhasználónév ékezetes karaktereket tartalmaz, ezek automatikusan eltávolítódnak

4. **"Erre a napra már van rögzített munkaidő"**
   - **Megoldás:** Ez normális működés - egy napra csak egy rögzítés lehetséges
