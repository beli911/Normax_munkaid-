## 🗄️ ADATBÁZIS STRUKTÚRA

### Táblák:

#### 1. `profiles` (Felhasználói profilok)
```sql
CREATE TABLE public.profiles (
  id uuid PRIMARY KEY REFERENCES auth.users(id),
  full_name text,
  is_admin boolean DEFAULT false,
  hourly_rate numeric DEFAULT 0,
  created_at timestamp with time zone DEFAULT now()
);
```

#### 2. `timesheets` (Munkaidő bejegyzések)
```sql
CREATE TABLE public.timesheets (
  id bigint PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  user_id uuid REFERENCES public.profiles(id) NOT NULL,
  work_date date NOT NULL,
  entry_type text CHECK (entry_type IN ('work', 'holiday', 'sick_leave')),
  status text DEFAULT 'pending' CHECK (status IN ('pending', 'approved', 'rejected')),
  start_time time,
  end_time time,
  note text,
  expense_amount numeric DEFAULT 0,
  expense_note text,
  is_company_card boolean DEFAULT false,
  payment_id bigint REFERENCES public.payments(id),
  created_at timestamp with time zone DEFAULT now()
);
```

#### 3. `payments` (Fizetési nyugták)
```sql
CREATE TABLE public.payments (
  id bigint PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  user_id uuid REFERENCES public.profiles(id) NOT NULL,
  total_paid numeric NOT NULL,
  payment_date timestamp with time zone DEFAULT now() NOT NULL,
  admin_id uuid REFERENCES public.profiles(id)
);
```

## 📊 SQL FÜGGVÉNYEK

### 1. `get_weekly_report(start_date, end_date)`

**Funkció:** Heti riport generálása (kifizetett + kifizetetlen)

**Visszatérési értékek:**
- `user_id` - Felhasználó ID
- `full_name` - Teljes név
- `hourly_rate` - Óradíj
- `total_hours` - Összes óra
- `total_wages` - Összes munkabér
- `total_expenses` - Kifizetendő költség (saját zseb)
- `company_expenses` - Céges költség (csak info)
- `grand_total` - Végösszeg (bér + saját költség)
- `is_fully_paid` - Teljesen kifizetve?
- `payment_date` - Fizetés dátuma

**Kulcs logika:**
```sql
-- Kifizetendő Költség (SAJÁT ZSEB)
COALESCE(SUM(
  CASE WHEN COALESCE(t.is_company_card, false) = false AND t.expense_amount > 0 
    THEN t.expense_amount ELSE 0 END
), 0)::numeric(10, 0) AS total_expenses,

-- Céges Költség (CSAK INFÓ)
COALESCE(SUM(
  CASE WHEN COALESCE(t.is_company_card, false) = true AND t.expense_amount > 0 
    THEN t.expense_amount ELSE 0 END
), 0)::numeric(10, 0) AS company_expenses,

-- Végösszeg: Bér + Csak a Saját Költség!
((COALESCE(SUM(...)) * COALESCE(p.hourly_rate, 0)) + 
COALESCE(SUM(
  CASE WHEN COALESCE(t.is_company_card, false) = false AND t.expense_amount > 0 
    THEN t.expense_amount ELSE 0 END
), 0))::numeric(10, 0) AS grand_total
```

### 2. `get_weekly_details_for_user(start_date, end_date, user_id)`

**Funkció:** Heti részletek egy felhasználóhoz

**Visszatérési értékek:**
- `work_date` - Munkanap
- `entry_type` - Bejegyzés típusa
- `description` - Leírás
- `hours` - Munkaóra
- `wage_amount` - Munkabér
- `expense_amount` - Költség
- `is_company_card` - Céges kártya?
- `total_daily` - Napi összesen
- `is_paid` - Kifizetve?
- `payment_date` - Fizetés dátuma

### 3. `execute_payment_until(user_id, amount, cutoff_date)`

**Funkció:** Fizetés rögzítése egy dátumig

**Kulcs logika:**
```sql
-- 1. Nyugta létrehozása
INSERT INTO public.payments (user_id, total_paid, admin_id)
VALUES (p_user_id, p_amount, v_admin_id)
RETURNING id INTO v_payment_id;

-- 2. Csak azokat a sorokat zárjuk le, amik a DÁTUMON BELÜL vannak
UPDATE public.timesheets
SET payment_id = v_payment_id
WHERE user_id = p_user_id
  AND payment_id IS NULL
  AND status != 'rejected'
  AND work_date <= p_cutoff_date;
```
