# Munkaidő Nyilvántartó Rendszer

Egy modern, mobilra optimalizált munkaidő-nyilvántartó webalkalmazás Next.js és Supabase használatával.

## 📖 Dokumentáció

- **👉 Részletes beállítási útmutató: [SETUP.md](./SETUP.md)**
- **🔧 Technikai dokumentáció és API referencia: [TECHNICAL-DOCUMENTATION.md](./TECHNICAL-DOCUMENTATION.md)**

A technikai dokumentáció tartalmazza az API endpointokat, adatbázis struktúrát, integrációs lehetőségeket és példa kódokat külső szoftverekkel való integrációhoz.

## 📂 Projekt struktúra

- **`app/`** – Next.js oldalak, API route-ok, komponensek
- **`lib/`** – Supabase kliens, segédfüggvények (utils)
- **`database/`** – SQL scriptek (setup, migrációk, fixek) – a Supabase SQL Editor-ban futtatandók
- **`scripts/`** – Segédprogramok (pl. `create-admin.js`, `check-admin.js`) – a projekt gyökeréből: `node scripts/create-admin.js`

## 🚀 Gyors Telepítés

### 1. Függőségek telepítése

```bash
npm install
```

### 2. Környezeti változók beállítása

Hozz létre egy `.env.local` fájlt a projekt gyökerében:

```env
NEXT_PUBLIC_SUPABASE_URL=ide_írd_a_supabase_url_t
NEXT_PUBLIC_SUPABASE_ANON_KEY=ide_írd_az_anon_key_t
```

A Supabase kulcsokat a [Supabase Dashboard](https://supabase.com/dashboard) → Project Settings → API menüpontban találod.

### 3. Adatbázis beállítása

1. Lépj be a [Supabase.com](https://supabase.com)-ra
2. Hozz létre egy új projektet
3. Menj az **SQL Editor** menüpontba
4. Futtasd le az alábbi SQL scriptet:

```sql
-- 1. PROFILOK TÁBLA (Hogy ne ID-kat láss, hanem neveket)
create table public.profiles (
  id uuid references auth.users on delete cascade not null primary key,
  email text,
  full_name text,
  is_admin boolean default false
);

-- 2. MUNKAIDŐ TÁBLA
create table public.timesheets (
  id bigint generated by default as identity primary key,
  user_id uuid references public.profiles(id) not null,
  work_date date not null default current_date,
  start_time time not null,
  end_time time not null,
  note text,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 3. AUTOMATIZMUS (Trigger)
-- Ha valaki regisztrál, hozzunk létre neki egy profilt automatikusan
create or replace function public.handle_new_user() 
returns trigger as $$
begin
  insert into public.profiles (id, email, full_name)
  values (new.id, new.email, new.raw_user_meta_data->>'full_name');
  return new;
end;
$$ language plpgsql security definer;

create trigger on_auth_user_created
  after insert on auth.users
  for each row execute procedure public.handle_new_user();

-- 4. BIZTONSÁG (Row Level Security)
alter table public.profiles enable row level security;
alter table public.timesheets enable row level security;

-- Szabályok Profilokhoz:
create policy "Mindenki láthatja a profilokat" on profiles for select using (true);
create policy "User módosíthatja a sajátját" on profiles for update using (auth.uid() = id);

-- Szabályok Munkaidőhöz:
-- A Dolgozó csak a sajátját látja és írhatja
create policy "Dolgozó látja a sajátját" on timesheets for select using (auth.uid() = user_id);
create policy "Dolgozó írhatja a sajátját" on timesheets for insert with check (auth.uid() = user_id);

-- Az ADMIN mindent lát
create policy "Admin mindent lát" on timesheets for select 
using (exists (select 1 from profiles where id = auth.uid() and is_admin = true));
```

### 4. Admin jogosultság beállítása

1. Regisztrálj az alkalmazásba a saját email címeddel
2. Menj vissza a Supabase **Table Editor** részébe
3. Keresd meg a `profiles` táblát
4. A saját sorodban pipáld be az `is_admin` mezőt

## 🏃 Futtatás

### Fejlesztési mód

```bash
npm run dev
```

Nyisd meg a böngészőben: [http://localhost:3000](http://localhost:3000)

### Éles környezet

A projekt könnyen telepíthető [Vercel](https://vercel.com)-re:

1. Regisztrálj a Vercel.com-ra
2. Kösd össze a GitHub fiókoddal
3. Töltsd fel a kódot
4. Add meg a környezeti változókat
5. Kész! 🎉
